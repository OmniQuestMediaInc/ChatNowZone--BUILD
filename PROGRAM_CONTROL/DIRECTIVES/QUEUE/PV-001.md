## PART B — SERVICE IMPLEMENTATIONS

### B1 — DiamondTierService (Module 1)

**File:** `services/core-api/src/dfsp/diamond-tier.service.ts` (CREATE)

Manages Diamond account tier lifecycle: eligibility gate, tier assignment,
demotion on risk escalation, and audit trail.

**Scope:**
- `isEligibleForDiamond(userId): Promise<boolean>` — checks KYC status, risk tier,
  no active account holds, no open chargebacks
- `assignDiamondTier(userId, agentId, correlationId)` — sets tier to DIAMOND,
  appends audit event, NATS event DIAMOND_TIER_ASSIGNED
- `demoteTier(userId, targetTier, reason, agentId, correlationId)` — downgrades tier,
  cancels any active Diamond quote sessions, NATS event DIAMOND_TIER_DEMOTED
- `getDiamondStatus(userId)` — returns current tier, hold status, active contracts
- All state changes append to AuditEvent (event_type: DIAMOND_TIER_CHANGE)
- NATS events: DIAMOND_TIER_ASSIGNED, DIAMOND_TIER_DEMOTED
- rule_applied_id on every output

### B2 — DiamondKycService (Module 2)

**File:** `services/core-api/src/dfsp/diamond-kyc.service.ts` (CREATE)

Enhanced identity verification layer for Diamond/VIP accounts, building on the
existing IdentityVerification schema. Adds biometric liveness check tracking,
document re-verification schedule, and Diamond-specific KYC hold gates.

**Scope:**
- `requiresDiamondKyc(userId)` — returns true if any of: KYC expired, liveness
  not passed, Diamond KYC not completed
- `recordDiamondKycPass(userId, verificationId, agentId, correlationId)` — marks
  Diamond KYC complete, sets next_review_due (12 months from now)
- `flagKycHold(userId, reason, correlationId)` — blocks Diamond purchases until
  KYC hold cleared; NATS event DIAMOND_KYC_HOLD_APPLIED
- `clearKycHold(userId, agentId, correlationId)` — requires supervisor approval;
  NATS event DIAMOND_KYC_HOLD_CLEARED
- All reads from existing IdentityVerification table (no new table)
- NATS events: DIAMOND_KYC_HOLD_APPLIED, DIAMOND_KYC_HOLD_CLEARED,
  DIAMOND_KYC_PASS_RECORDED
- rule_applied_id on every output

### B3 — IntegrityHoldService (Module 11)

**File:** `services/core-api/src/dfsp/integrity-hold.service.ts` (CREATE)

Manages the escrow-style integrity hold that is established at contract
payment initiation and captured (not released) on chargeback. The hold amount
is set per contract and may differ from the CARF amount over time.

**Scope:**
- `applyHold(contractId, userId, holdAmountCents, correlationId)` — creates
  integrity_holds record with status HELD; NATS event INTEGRITY_HOLD_APPLIED
- `captureHold(contractId, captureAmountCents, correlationId)` — on chargeback:
  transitions to CAPTURED (or PARTIALLY_CAPTURED), appends ledger CAPTURE entry
  (append-only); NATS event INTEGRITY_HOLD_CAPTURED
- `releaseHold(contractId, correlationId)` — on tokens_released status:
  transitions to RELEASED, appends ledger HOLD_RELEASE entry;
  NATS event INTEGRITY_HOLD_RELEASED
- `getHoldStatus(contractId)` — returns current hold record
- Ledger entries are append-only (CAPTURE / HOLD_RELEASE entry types)
- NATS events: INTEGRITY_HOLD_APPLIED, INTEGRITY_HOLD_CAPTURED,
  INTEGRITY_HOLD_RELEASED
- rule_applied_id on every output

### B4 — ComplianceReportingService (Module 15)

**File:** `services/core-api/src/dfsp/compliance-reporting.service.ts` (CREATE)

Generates CARF (Chargeback Administration and Recovery Fee) capture reports and
FINTRAC-style large transaction threshold reports for regulator submission.
All reports are read-only aggregations — no ledger writes.

**Scope:**
- `generateCarfReport(periodStart, periodEnd, correlationId)` — aggregates all
  CARF amounts captured in period from integrity_holds + diamond_contracts;
  outputs structured JSON + CSV; NATS event CARF_REPORT_GENERATED
- `generateFintracThresholdReport(periodStart, periodEnd, correlationId)` — aggregates
  transactions >= GovernanceConfig.FINTRAC_REPORTING_THRESHOLD_CAD across period;
  outputs FINTRAC-compatible JSON; NATS event FINTRAC_REPORT_GENERATED
- `getReportHistory(reportType, limit)` — returns prior report metadata from audit_events
- All report generation appends to AuditEvent (event_type: COMPLIANCE_REPORT_GENERATED)
- Report content stored by reference (file path/URL) — not inline in DB
- NATS events: CARF_REPORT_GENERATED, FINTRAC_REPORT_GENERATED
- rule_applied_id on every output
